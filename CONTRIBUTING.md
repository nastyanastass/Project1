# Інструкція для роботи з проєктом (CONTRIBUTING.md)

## Для користувачів (як запустити програму)
1. Завантажте проєкт
2. Відкрийте папку проєкту
3. Запустіть файл:

```bash
python Program.py
```

---

## Для розробників

### Клонування репозиторію
```bash
git clone https://github.com/nastyanastass/Project1.git
```

---

### Створення нової гілки
```bash
git checkout -b feature/нова-функція
```

Рекомендується використовувати:
- feature/
- fix/
- docs/
- refactor/
- test/

---

## Внесення змін
Після редагування коду:
```bash
git add .
git commit -m "feat: додано новий функціонал"
```

---

## Відправлення змін
```bash
git push origin feature/нова-функція
```

---

## Створення Pull Request
1. Відкрити GitHub
2. Натиснути New Pull Request
3. Обрати потрібну гілку
4. Описати зміни

---

## Правила стилю коду
- Дотримуватись стилю Python
- Використовувати зрозумілі назви змінних
- Додавати коментарі
- Не видаляти існуючі тести

---

## Тестування
Перед відправленням змін необхідно перевірити роботу тестів.

### Assert тести
```bash
python test_simple.py
```

### unittest
```bash
python test_unit.py
```

### pytest
```bash
pytest -v
```

---

## Додавання нових тестів
Нові тести потрібно додавати у:
- test_simple.py
- test_unit.py
- test_pytest.py

---

## Можливі покращення
- Додати нові типи конвертації
- Покращити дизайн
- Додати темну тему
- Розширити тестове покриття

---

## Репорти про помилки
Якщо знайдена помилка:
1. Відкрити вкладку Issues
2. Натиснути New issue
3. Описати проблему
4. За можливості додати скріншоти
